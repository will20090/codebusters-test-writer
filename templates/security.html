Privacy & Security

Understandably, there are concerns about the confidentiality of test content. Here is how all data is stored and protected on this site.

Methods

Encryption of Questions
All test questions and content are encrypted before being stored in the database using Fernet symmetric encryption (AES-128-CBC with HMAC-SHA256). This means that even though your test data is stored in a database, and somebody can see it, the data is impossible to decipher.
Encryption Key
The encryption key is stored as a server-side environment variable and is never exposed to the client or stored in the database. It is only accessible to the server process at runtime.
Access
Each test is tied to a specific user account. Only the owner and explicitly shared collaborators can access a test. All API endpoints require authentication and verify ownership before returning any data.
Passwords
Passwords are never stored in plaintext. They are hashed using werkzeug.security (PBKDF2 + SHA256) before being saved to the database.

Source Code

This project is open source. You can review and verify all the encryption methods listed above: https://github.com/will20090/codebusters-test-writer.
